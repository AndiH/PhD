Function for fitting a sum of two Gaussians (not coupled)

Fits the sum of two Gaussian distributions on a histogram. Can try to determine the ranges automatically by pre-fitting two individual Gaussians in, each, +- 10 percent of the center (inner), and +- 80 percent of the center (outer). Or, ranges of the inner and outer fits can be given. There's a dedicated function to specify the parameters of the double Gaussian fit manually, which is also called by this function.

Parameters
histHistogram with the data to fit to
verbosePrint fitting parameters
useAutoRangeDefined the ranges of the pre-fits automatically to be 10% and 80% around zero
innerRangeMaxThe positive half of the range of the inner Gauss pre-fit
outerRangeThe positive half of the range of the outer Gauss pre-fit. For non-zero-centered histograms, please use the manual function
Returns
A TF1 with a double Gaussian. First three parameters are the parameters of the inner Gaussian, second set of three parameters are those of the outer.

THStack* andi::histogramsToStack ( TObjArray *  histos,
TString  stackAddOption = "" 
)

Converts an array of histograms (as TObjArray) into a THStack.

Gives the stack proper name, tile, and axes on the way, so it can be manipulated as a usual ROOT object (because usually, it can't, as the stuff is only defined after drawing).

The histograms are provided as a list in TObjArray to be more flexible and not deal with the overhead of more-complex constructors. At least for the time being the flexibilty is not made use of since only TH1D are used.

For ROOT to generate axes, the histogram first needs to be drawn. In theory, "goff" should not print the canvas on your screen. Instead, a kind of empty canvas is drawn.

Parameters
histosTObjArray with histograms. Take care of the order!
stackAddOptionOptional string which is used when stack->Add(hist, option)
Returns
THStack of histograms

Reads in a Tree from multiple Files.

For data collections of the same structure it is possible to append the trees after one another. This functions gives a convenient way to do so.

The fileName parameter should point to a text file with the file name of a root file per line, like:

file1.root
file2.root

The function returns a TTree with combined content.

Parameters
treeNameName of the tree in every file. Has to be equal.
fileNameA text file with one file per line.
Returns
TTree of a chain of input files. The intermediate TChain is casted to a TTree, because of my macros.
